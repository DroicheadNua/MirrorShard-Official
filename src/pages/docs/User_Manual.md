## AI機能の使い方  
MirrorShardには現在、メインウィンドウで運用する統合型AI機能と、別ウィンドウで開くAIチャット機能の、２つのAI機能があります。    
詳しい使い方は「AI機能使用ガイド」を参照してください。  

## エクスポート  
Ctrl + Eでエクスポートウィンドウを開くことができます。  
このウィンドウでは横書きの出力にのみ対応しています。縦書きでエクスポートしたい場合は、縦書きプレビューからエクスポートしてください。  

### 機能と操作  
* 印刷/PDF保存: 横書きでPDFファイルをエクスポート、またはプリンタで印刷します。ルビにも対応しています。  
* HTML保存: 横書きでHTMLファイルをエクスポートします。  
* EPUB保存: 横書きでEPUBファイルをエクスポートします。  
* フォント: フォントを選択します。明朝（Serif）、ゴシック（Sans-serif）、等幅（Monospace）から選択できます。  
* フォントサイズ: フォントサイズを設定します。  
* 行間: 行間を設定します。  

## 縦書きプレビュー  
青空文庫形式のルビに対応した縦書きプレビューです。
ここから縦書きでエクスポート（HTML、EPUB）することも可能です。  
※縦書きエクスポートにはPandocが必要です。導入方法は「Pandoc導入ガイド」を参照してください。  
※縦書きでの印刷、及びPDF保存はできません。HTMLで保存し、ブラウザ等で表示・印刷してください。  

### 機能と操作  
* エクスポート: 縦書きでエクスポート（HTML、EPUB）します。ルビにも対応しています。
* 更新: プレビューを更新します。  
* 最前面に表示: プレビューウィンドウを常に最前面に表示します。  

## Markdown / HTML プレビュー  
エディタのUIボタン、または Ctrl + M (Cmd + M) でプレビューウィンドウを開きます。  
上部のプルダウンから「Markdownモード」と「HTMLモード」を切り替え可能です。  

### 機能と操作  
* ズーム: Ctrl + マウスホイール、または Ctrl + +/- で表示倍率を変更できます。Ctrl + 0 でリセットします。  
* アウトライン連動: メインエディタのアウトラインをクリックすると、プレビュー側も該当する見出し位置まで自動スクロールします。  
* HTMLソースをコピー: 現在表示されている内容を、HTMLタグを含めた状態でクリップボードにコピーします。ブログへの貼り付け等にご活用ください。  
* HTMLとして保存: 現在表示されている内容を、HTMLファイルとして保存します。
* DevToolsを開く: 開発者ツールを開きます。  
* ブラウザで開く: 「ブラウザで開く」アイコンをクリックすると、現在編集中のファイルをOS標準のブラウザ（ChromeやEdge等）で開きます。※ファイルを保存してから実行してください。  
* 最前面に表示: プレビューウィンドウを常に最前面に表示します。
* 更新(Ctrl+R): プレビューを更新します。  

### 仕様と制限事項  
* **レイアウトの再現性:** プレビューウィンドウは簡易的なブラウザエンジンを使用しているため、古い記述法のCSSや、特定の画面幅を要求するレイアウトなどは正しく表示されない場合があります。これらも「ブラウザで開く」機能でご確認ください。  
* **リンクの挙動:** プレビュー内のリンクをクリックした際、Webサイト（http/https）へのリンクは標準ブラウザで開かれます。ローカルファイルへのリンクは、MirrorShardの新しいタブとして開かれます。  
* **外部スクリプト:** セキュリティ上の理由（CSP）により、プレビュー内での外部JavaScriptの実行や外部CSSの読み込みは制限されています。  
* **巨大テキスト:** 50,000文字を超えるテキストをプレビューする場合、パフォーマンス維持のため先頭部分のみが表示されます。  
* コードブロック: 極めて長いコードブロックや、特殊な記号を含むソースコードをMarkdownとして表示する場合、レンダリングが途切れたり崩れたりすることがあります。その場合は「コードエディタモード（Ctrl+K）」での編集を推奨します。  

### Webフレームワーク（Astro/Next.js等）のプレビューについて  
HTMLプレビュー機能は、静的なHTMLファイルの確認を想定しています。  
AstroやReactなどで使われる動的なパス解決（public フォルダへのルートパス参照や、import 構文など）は、プレビュー画面では正しく処理されない（画像が表示されない、リンク先にジャンプできない等）場合があります。  
これらのプロジェクトをプレビューする場合は、内蔵ターミナル（Ctrl + @） を開き、npm run dev 等でローカルサーバーを立ち上げて、ブラウザで確認することを推奨します。  

## コードエディタモード  
Ctrl + Kでコードエディタモードに切り替えることができます。  
いたってシンプルな作りで、対応言語はHTML/CSS/JavaScript・Markdown・TypeScript・Rust・Pythonのみとなっています。  
機能としてはシンタックスハイライトに加え、以下の入力支援機能が利用可能です。  
・オートインデント  
・コード折りたたみ（Code Folding）  
・ブラケットの自動補完（Auto Close Brackets）  
・キーワード補完（簡易オートコンプリート）  

### 簡易フォーマッタ (Alt + Shift + F)  
ファイル全体のインデント（字下げ）を自動的に整えます（コードエディタモード時のみ有効）。  
※内部的に「全選択→整形」の処理を行っているため、実行後に「元に戻す（Undo）」を行うと、一時的にテキストが全選択された状態に戻る場合があります。仕様となりますのでご了承ください。  

### AIコード補完（Alt + Enter）  
コードエディタモード時にAlt + Enterを押すと、AIによるコードの補完を行います。詳しくは「AI機能使用ガイド」を参照してください。  

## 内蔵ターミナル (Terminal)  
`Ctrl + @` で、エディタ上に半透明のターミナルウィンドウを開閉できます。  
執筆中に Git コマンドを実行したり、Node.js プロジェクトを操作したりするのに便利です。  

*   **シェル**: Windowsでは PowerShell (または Git Bash)、Mac/Linuxではデフォルトのシェルが起動します。設定画面でパスを変更することも可能です。  
    *   ※Git Bashを使用する場合は `bin\bash.exe` を指定してください。  
    *   ※Git Bash使用時、pnpm のシンボリックリンクとの相性によりエラーが発生する場合があります。回避策についてはFAQをご覧ください。  
*   **Open Terminal Here**: エディタのタブを右クリック（またはコンテキストメニュー）して「Open Terminal Here」を選択すると、そのファイルがあるディレクトリでターミナルを開きます。  
*   **フォント連動**: 「コードエディタモード」のフォント設定がターミナルにも適用されます。
*   exitコマンドを入力することで、ウィンドウを閉じることができます。  


## 検索ウィンドウの使い方  
Ctrl + F（MacはCmd + F）を押すことによって、検索ウィンドウを開くことができます。  
パネルの各機能は以下の通りです。  

Find: 検索したいキーワードを入力します。  
Replace: キーワードを置換したい場合、置換後のキーワードをここに入力します。  
next / previous: 次/前のマッチ箇所へジャンプします。  
all: すべてのマッチ箇所を選択します。  
ショートカットにも対応しています。  
　next: F3 (Mac: Cmd+G)  
　previous: Shift+F3 (Mac: Cmd+Shift+G)  
replace / replace all: 現在のマッチ箇所を置換 / すべてのマッチ箇所を置換します。  
match case: チェックを入れると、大文字と小文字を区別して検索します。（例: Bookとbookを別の単語として扱います）  
regexp (チェックボックス): チェックを入れると、正規表現による高度な検索が有効になります。（例: ^#で行頭の見出しを検索するなど）  
by word (チェックボックス): チェックを入れると、単語単位で完全に一致するものだけを検索します。（例: catを検索したときに、categoryの中のcatにはマッチしなくなります）  
閉じる: 検索パネルは、ESCキーを押すか、パネル右上の×ボタンを押すことで閉じることができます。  

## ステータスバー  
　画面下部のステータスバーには、左から順に  
* 現在開いているアウトライン項目  
* 現在開いているファイルのパス  
* カーソル位置の行番号：列番号／総行数  
* 総文字数  
* 選択中のファイルのエンコードと改行コード  
* 日付と現在時刻  
が表示されます。  

## 設定  
　F2キー、または画面上部の歯車アイコンを押すことにより、設定画面を開くことができます。  

■＜一般＞タブ  
◯エディタの最大幅……エディタ（文字が書ける部分）の最大幅を設定します。  
◯行の高さ……行間隔を調整します。  
◯エディタ左右余白……エディタの左右余白を調整します。  
◯禁則処理……禁則処理の厳密さを設定します。  
◯背景画像……任意の背景画像を読み込みます。背景画像はライトモードでのみ有効となります。  
◯BGM……任意のBGMを読み込みます。  
◯折り返し設定……単語（欧文）の途中で折り返すかどうかを設定します。  
◯フォント……お使いのPCからフォントを読み込みます。  
◯Pandoc Path……Pandocのパスを指定します。標準インストールした場合は空欄のままで大丈夫です。Pandocは、縦書きでエクスポートするときに必要になります。  
　■エディタ外観  
背景色や不透明度、すりガラス効果の強度など、エディタの外観を自由にカスタマイズできます。  
また、設定画面の「テーマプリセット」から、標準の「Default」「Paper」「Tokyo Night」などのテーマを選択できます。
また、ご自身でカスタマイズした外観を、ユーザーテーマとして作成・保存・削除することができます。  

■＜AI設定＞タブ
・Google Gemini  
◯API Key……GeminiのAPIキーをここに入力することにより、Geminiとの会話が可能になります。  
◯Model……Geminiのモデルを指定します。使用するモデルと料金プランによっては課金が発生する場合がありますのでご注意ください。  
・Local LLM  
◯EndpointURL……使用するLLMのAPIエンドポイントURLを指定します。右側のプルダウンから、LM StudioとOllamaを使用する際のデフォルト値を読み込むことが出来ます。  
◯Model Name……使用するモデルの名前を指定します。Ollamaでは必須ですが、LM Studioではこの値は無視されます（モデルはLM Studio側で指定するため）  
・Behavior  
◯Max Output Tokens……AIの応答の長さを指定します。日本語の1文字は、およそ2トークンとして計算されます。  
**※Gemini 2.5系などを使用する場合は、内部的な思考プロセスでトークンを消費することがあるため、大きめの値を設定することを推奨します。**  
◯AI思考時の演出を有効にする……AIが思考している間の演出（オーバーレイ）の有無を切り替えます。  
◯System Prompt……システムプロンプトを設定します。  
・User Profile  
◯Name……チャット欄に表示されるユーザーの名前を設定します。  
◯Icon……チャット欄に表示されるユーザーのアイコンを設定します。  
・AI Profile  
◯Name……チャット欄に表示されるAIの名前を設定します。  
◯Icon……チャット欄に表示されるAIのアイコンを設定します。  

■＜コードエディタ＞タブ  
・Code Editor Settings  
◯Syntax Highlight Language……コードエディタモードでのシンタックスハイライトに使用する言語を指定します。通常はファイルの拡張子によって自動判別されます。  
現在、HTML・CSS・Javascript/Typescript・Markdown・Rust・Pythonにのみ対応しています。  
◯Font……コードエディタモードでのフォントを指定します。  
◯Font Size……コードエディタモードでのフォントサイズを指定します。  
◯Word Wrap（折り返し）……エディタ幅で折り返すかどうかを指定します。  
・Terminal Settings  
◯Shell Path……使用するシェルを指定します。空欄の場合、Windowsでは PowerShell (または Git Bash)、Mac/Linuxではデフォルトのシェルが起動します。  
	※Git Bashを使用する場合は `bin\bash.exe` を指定してください。  
◯Startup Directory……ターミナルを開く標準のフォルダを指定します。空欄の場合、ユーザーのホームフォルダになります。
